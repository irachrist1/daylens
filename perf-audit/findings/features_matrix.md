# Daylens Feature Matrix & Performance Audit

Comprehensive map of user-facing modules, background systems, and in-progress work. **Works?** reflects current code behavior, not marketing intent. Performance notes tie to `composer.md` / codebase reads.

| Feature Name | Shipped Status | Works? | Performance Bottlenecks | What Could Be Done Better | Does It Matter? |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Always-on foreground app tracking** (`tracking.ts`) | Shipped | Yes (macOS native, Windows, Linux fallbacks) | 5s poll; session flush invalidates timeline/apps/insights scopes; Linux uses sync subprocess calls (`hyprctl`, `swaymsg`, `xdotool`, `xprop`) that can block main thread. | Debounce projection invalidation; async Linux polls; incremental today projection instead of full rebuild on flush. | **High** — core data pipe; every downstream feature depends on it. |
| **Native focus-event capture helper** (`focusCapture.ts`, `capture-helper`) | Shipped (macOS; build required) | Yes when binary present | One SQLite INSERT per helper event; no batching; dense tab/window churn = WAL churn. | Batch inserts in transactions every 250ms; feeds chunk2 projections off hot path. | **High** — powers derived sessions/blocks and finalize-at-startup. |
| **Live session snapshot & crash recovery** (`tracking.ts`) | Shipped | Yes | Recovery flush triggers same triple invalidation as normal flush. | Same as tracking invalidation debounce. | **Medium** — correctness on restart; rare path. |
| **Idle / away / lock / suspend activity events** (`tracking.ts`, `activity_state_events`) | Shipped | Yes | Gap-boundary queries in `gapHasHardActivityBoundary` per inter-session gap during block build. | Prefetch day’s activity events once for segmentation. | **Medium** — improves timeline gap accuracy; not on every tick. |
| **Browser history import** (`browser.ts`) | Shipped | Yes (Chromium family, Firefox, Arc, Dia, Comet, etc.) | Every 60s: sync `copyFileSync` of each profile History DB + open SQLite on main thread. | Async copy; worker thread; incremental WAL read; cache profile list. | **High** — website context for timeline; sync copy pauses entire app. |
| **Active browser tab context** (`browserContext.ts`) | Shipped | Yes | On tracking poll when browser foreground: sync `osascript` (1.5s timeout) or Chromium history copy per sample. | Lower sample rate; cache last tab; move to helper process. | **High** — page-level timeline evidence; runs every 5s in browser-heavy use. |
| **Windows ActivityCache backfill** (`windowsHistory.ts`) | Shipped (Windows) | Yes | 5s after launch: up to 2000 rows inserted without dedup guard; may repeat. | Backfill cursor + INSERT OR IGNORE on (bundle_id, start_time). | **Medium** — one-time/history fill; duplicate risk matters for data quality. |
| **iMessage capture** (`imessageCapture.ts`) | Shipped, opt-in (Settings) | Partial (macOS + Full Disk Access) | Runs immediately at startup when enabled; reads chat.db synchronously. | Defer first sync like browser; async permission checks. | **Low** — off by default; Apple-gated; niche. |
| **File activity events** (`file_activity_events` schema) | Not shipped (schema only) | No UI / no writer | Table exists; used only if populated — `workMemory.gatherConcurrentEvidence` queries it when rows exist. | Implement capture or remove dead schema from hot paths. | **Low** until capture ships — infrastructure only. |
| **Entity suggestions** (`entity_suggestions` schema) | Not shipped | No UI | Table + indexes only; no renderer or IPC surface found. | Ship UI or drop from migration path. | **Low** — future attribution assist. |
| **Timeline day view** (`Timeline.tsx`, `GET_TIMELINE_DAY`) | Shipped | Yes | Today never uses persisted-block cache; full `getTimelineDayPayload` + `persistTimelineDay` on every read; 30s poll; N+1 in persisted loader; large IPC payload. | Cache today incrementally; bulk-fetch block members/labels; slim IPC mode; stop write-on-read. | **High** — primary screen. |
| **Timeline week view** (`Timeline.tsx` WeekView) | Shipped | Yes | 7× parallel `getTimelineDay` every 30s when week includes today; week AI review extra IPC. | Single week-summary IPC; poll only today. | **High** — common navigation mode. |
| **Timeline block inspector** (label override, AI regen, client assign) | Shipped | Yes | `listClientsDetailed` refetched on every block selection; `regenerateBlockLabel` ships full block over IPC; inline row re-renders. | Fetch clients once per mount; pass blockId not full block; memoize rows. | **High** — core editing loop on timeline. |
| **Block label overrides** (`SET_BLOCK_LABEL_OVERRIDE`) | Shipped | Yes | Triple projection invalidation on save; triggers work-memory learning write. | Invalidate only timeline scope for label edits. | **Medium** — infrequent user action. |
| **Timeline rebuild / AI re-analyze day** (`REBUILD_TIMELINE_DAY`) | Shipped | Yes | Sequential `await generateWorkBlockInsight` per eligible block inside IPC handler. | Background job + progress events; return immediately. | **Medium** — manual power-user action. |
| **Timeline segmentation & gap groups** (`workBlocks.ts`) | Shipped | Yes | Full recompute path for today; `buildBlockFromCandidate` re-queries websites per block. | Day-level website prefetch; persist today only on material change. | **High** — defines what users see. |
| **Derived projections (chunk2)** (`chunk2.ts`, `projectDay`) | Shipped | Yes | Full-day `focus_events` scan + delete/reinsert derived tables; startup finalize + stale-day sweep (up to 7 days); extra COUNT re-scan. | Stream/chunk events; background worker; skip COUNT when events already loaded. | **High** — backs past-day fast path and sync. |
| **Write-on-read derived day open** (`projections.ts` getDerivedDayTimelinePayload) | Shipped | Yes | Rebuilds + `persistTimelineDay` on every past-day read when derived data exists. | Read-only serve from derived + persisted; persist only on edit. | **High** — every historical navigation hit. |
| **getBlockDetail 31-day scan** (`workBlocks.ts` GET_BLOCK_DETAIL) | Shipped | Yes | Worst case 31× full `getTimelineDayPayload` per lookup. | Direct SQL lookup by block id. | **Medium** — rare unless UI calls it often. |

## Summary: where perf work matters most

| Priority | Features | Why |
| :--- | :--- | :--- |
| **P0** | Timeline day/week, projection invalidation, tracking flush, derived write-on-read, startup DB repair | On every launch, every 5–30s, every app switch |
| **P1** | Browser history/context, attribution pipeline, AI tab load/recap, Apps digest, sync snapshot | Heavy main-thread or IPC under normal use |
| **P2** | Search, icons, Settings mount, Day Wrapped, block inspector, lazy routes | Noticeable but situational |
| **P3** | MCP, iMessage, process monitor, snapshot export, entity/file activity (unshipped) | Off by default, rare, or not wired |

## Not shipped but present in codebase

- **AI Spend Panel** — component + IPC ready; not placed in Settings UI.
- **File activity capture** — DB schema only.
- **Entity suggestions** — DB schema only.
- **Dedicated Clients / Focus / History routes** — redirected to Timeline; APIs exist.
