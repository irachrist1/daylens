10 · Dead-end registry — where a user gets stuck with no feedback

  • verifying_permission (mac): no button; recovery rides on a 650ms timer whose IPC errors are uncaught. Onboarding.tsx:1309, 755-759, 698-746
  • proof stage: Continue disabled until data appears, no skip; all three data calls swallow failures — “Waiting for the first signal…” can last forever. Onboarding.tsx:1340, 788-790
  • Linux proof: tells you to fix Capture health “after setup” while blocking setup on the thing it can't capture. Onboarding.tsx:1371-1373
  • Silent report no-op: notifications enabled but no AI provider → nothing ever fires, nothing says why. dailySummaryNotifier.ts:112-130
  • Shipped crash black hole: renderer error → ErrorBoundary fallback UI, reported nowhere (no componentDidCatch; no keys in release workflows). ErrorBoundary.tsx:6-9
  • Checkout without checkout: billing unavailable removes the buttons but shows no explanation of what Plus even is. Settings.tsx:1411, 1438
  • Usage CSV export: fire-and-forget; no success or failure feedback. Settings.tsx:1958
  • persistOnboarding failures: most stage transitions are void-called; a failed settings write leaves the user on the same screen with no message. Onboarding.tsx:685-696
  • DB corruption (from the audit): fatal crash loop, no integrity check, no repair path — the terminal dead end. database.ts:46-124
