# Architecture Decision Records

One file per decision that is both hard to reverse and surprising. If a decision is easy to undo or obvious, it does not need an ADR.

Format. A number, a short title, the date, a status (proposed, accepted, superseded), then a short paragraph covering what we decided, why, and what we rejected. Write it once and do not edit it after it is accepted. If the decision later changes, add a new ADR that supersedes it. The staleness is the point. An ADR records what was true when we decided, not what is true today.

Numbering is sequential. 001, 002, and so on.
